# M6 — rails-era skin + re-theme moment (2026-07-05)

Seventh build session. Goal: the engineering contract goes on stage — a
second real skin (`rails-era`, the 2014 product web) lands as one JSON file
plus a rebuild with zero component edits, and Chapter 1's entry re-themes
the entire site to it as a designed moment: a GSAP-choreographed
custom-property swap, instant under reduced motion, that never touches the
visitor's own skin choice.

## Session shape

- One frontier orchestrator + two mid-tier agents + one small agent + one
  mid-tier verify agent, in three waves. The orchestrator landed everything
  taste-bearing or contract-bearing first: the era fonts, the full token
  set with a numeric WCAG audit, and the re-theme boundary component with
  its persistence contract encoded in unit tests — then the parallel wave
  built against pinned interfaces (the same fallback-as-base shape M5
  proved: the base component is fully functional with a stub motion module;
  the motion agent replaced only the stub's internals).
- **The skin is a Bootstrap-3-derived synthesis, audited before commit.**
  White body, gray chrome, hard 2–3px corners, 14px Helvetica-era body on
  Source Sans 3 with Source Code Pro mono (both OFL variable fonts,
  latin-subset, self-hosted, preloaded — Open Sans, the other obvious 2014
  face, is Apache-licensed and §3 pins OFL only). The status hues are the
  actual Bootstrap 3 alert text/background pairs, which turn out to pass
  AA as designed pairs (4.53–4.57:1). Two colors were corrected away from
  era-authentic values by the audit: the accent went from `#337ab7` to
  `#2c689c` so links clear 4.5:1 on every surface including wells, and the
  strong border went from `#adadad` to `#808080` to clear the 3:1 non-text
  bar galenti holds. The motion tokens are jQuery-era quad beziers at
  snappier durations — the skin's whole feel changes with zero component
  knowledge.
- **The re-theme boundary is a persistence contract first, a moment
  second.** `EraRetheme` saves the visitor's skin at entry, applies the era
  skin with `persist: false`, and restores on exit — but an explicit
  mid-chapter switch away wins (no restore), an explicit mid-chapter
  re-pick of the era skin itself wins (storage-delta rule), a stale stored
  preference from before the chapter never hijacks the restore, and
  localStorage is never written by story code. Eleven unit tests encode
  those rules; the e2e suite proves them in a real browser.
- The choreography stages an honest mechanism instead of hiding it: a 3px
  accent sweep travels the viewport in ~420ms, the `data-skin` flip fires
  at one-third travel (one frame, whole page), and the chapter settles with
  a 240ms transform/opacity stagger. The sweep bar is styled with
  `var(--color-accent)` so it re-tokens itself mid-travel — sienna above
  the flip, Bootstrap blue below it. Components that carry hover
  transitions crossfade their colors over ~160ms while structure snaps;
  that accidental micro-crossfade lands exactly where the eye rests and
  was kept as a feature. Reduced motion takes an instant swap and never
  downloads the motion chunk.

## Verify

- **The verify agent's one "failure" was the suite catching its own blind
  spot, and the agent misread it.** The full run came back 121/122 with one
  axe failure on `/story/rails-era` under the galenti skin, which the agent
  labeled pre-existing — impossible, since CI was green at M5. Orchestrator
  reproduction showed the truth: the shell-level axe matrix scans at
  nav-visible, which now lands mid-entrance — the settle stagger holds
  sections at 0.85 opacity, and axe grades galenti's marginal grays through
  the alpha blend (3.67:1 readings on annotation text). The settled page
  scans clean, zero violations. The fix was test architecture, not color
  and not an exclusion: the chapter re-themes itself, so a galenti/debug
  steady state no longer exists on that route — its axe coverage moved to
  the re-theme suite, which waits for the entrance deterministically (the
  last stagger target reaching opacity 1, no fixed sleeps). The same race
  would have made the debug-skin variant flaky; both entries retired with
  a comment naming the reason.
- The mid-settle readings also re-confirmed the M5 finding from the other
  side: galenti's muted/faint grays sit close enough to the AA line that
  any alpha blend tips them. The token-level question
  (`--color-ink-muted` at 4.24:1) is on the design-checkpoint agenda
  unchanged; rails-era was built with margin (muted 7.0:1) so the era skin
  has no such edge.
- **§9.3 re-theme capture wired and PASS, all four targets** (orchestrator
  re-ran after the agent for the record): assembly 509 frames / worst
  18.7ms, rail 388 / 18.7ms, palette 53 / 18.1ms, era re-theme 87 / 18.2ms
  — zero missed frames (>25ms), zero bursts, median-of-3 at 4× CPU
  throttle. The capture's era path is the real one: palette-driven Inertia
  visit from `/work` into the chapter, frame collector alive across the
  client-side navigation. The pending list is down to the M8 WebGL swap.
- The re-theme e2e suite (12 tests) proves the persistence contract in a
  real browser: entry flips `data-skin` and `--color-surface` measurably
  differs; exit restores; localStorage stays untouched through a full
  cycle; an explicit palette switch mid-chapter survives exit with the
  choice persisted; a `?skin=rails-era` deep link enters silently and
  exits still era-skinned; reduced motion swaps instantly with a
  network-level proof that no motion/gsap chunk is ever requested (with a
  motion-mode control on a fresh page so the cache can't lie); the
  `role="status"` region announces "Theme: Rails era"; and the sweep never
  traps keyboard focus.
- Full suite at close: **582 unit** · **84 request specs** · **120 e2e**
  (16-component axe matrix now runs galenti × debug × rails-era). Token
  discipline, RuboCop/ESLint/stylelint/Prettier/tsc, manifest drift gate,
  leak gate: all green.
