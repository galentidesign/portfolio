# M5 — assembly opening + prologue (2026-07-05)

Sixth build session. Goal: the site's thesis stated without words — raw
tokens assemble into a Button, a Form Field, organisms, and finally the
page's own shell, in a GSAP-pinned scroll sequence at `/`, handing off to a
prologue beat (2004–2013) and the chapter gateway. Reduced motion gets the
same narrative as a static stepped diagram; a skip control is visible from
frame one; the budget is 60fps at 4× CPU throttle, and a beat that can't
hold it gets cut before the budget does.

## Session shape

- One frontier orchestrator + one frontier motion agent + two mid-tier
  agents, sequenced: the orchestrator landed the storyboard (beat-by-beat
  table with scroll ranges and choreography notes), the timeline contract
  (`mountAssemblyMotion` — the v1.5 WebGL swap surface), and the full static
  base before any agent launched; then the motion layer (frontier) and the
  prologue (mid) ran in parallel; the e2e/perf verify agent ran last against
  the assembled page.
- **Fallback-as-base, decided at the architecture level:** the static
  stepped diagram is the component's base render — same DOM, two
  presentations. Content parity with reduced motion is structural (the e2e
  asserts the figcaption sets are identical across modes) instead of a
  second implementation kept in sync by discipline. GSAP loads by dynamic
  import only when motion is allowed: reduced-motion visitors never
  download animation code at all, which is both the a11y-honest and the
  Lighthouse-honest shape.
- The spec tags the reduced-motion fallback [M]; fallback-as-base absorbed
  it into the frontier base build by construction, and that mid-tier
  capacity moved to the verify agent. Nothing ran below its spec tier — the
  motion layer itself ran on a frontier-class agent against the pinned
  storyboard, with orchestrator review beat by beat.
- The prologue's era labels are J's own words, supplied at check-in and
  landed verbatim. The fourth (2012–2017, university teaching) deliberately
  overruns the band's 2013 edge into the Rails era — rendered as a dashed
  accent thread that continues past the rail, a parallel track rather than
  an error.

## Task log

- **Two component-level catches in orchestrator review of the live
  sequence:** the sticky skip control tucked itself under the sticky site
  bar (a spec violation — "visible from frame one"), fixed by clearing the
  bar's height; and the motion mode was rendering beats inside the static
  diagram's bordered figure cards, which read as a slide deck rather than
  material assembling on the page — under `data-motion` the exhibit chrome
  now drops and elements float on the surface, which also makes the finale
  (the facsimile bar translating up into the real shell bar) land as
  designed.
- **Two storyboard flourishes were simplified by the motion agent to hold
  the perf rules, both correctly:** the molecule beat's focus-ring pulse
  became a single opacity flash (the real ring is a CSS `outline` — a paint
  property the transform/opacity-only rule forbids animating), and the atom
  convergence collapses the chip _container_ inward rather than flying five
  chips on individually measured paths (concurrency stays ≤ ~10 nodes per
  beat, no per-frame layout reads).
- The hero never dims: it rides slightly proud (22px / 3.5% scale) through
  the whole sequence and only settles home in the shell beat — the name is
  the one persistent element the assembly happens around, in both modes,
  which also keeps the existing smoke test honest with zero special-casing.
- React 19's hooks lint caught a latest-callback ref being assigned during
  render in the base component; the assignment moved into an effect. The
  stylelint modern-notation rules reformatted the media query and the
  spacing-ramp rules — auto-fixed.
- **A wrong-working-directory incident burned twenty minutes and briefly
  looked like file corruption:** a `cd` into the private command-center
  repo for the previous milestone's work-log entry persisted into this
  session (the shell keeps cwd between calls), so the GSAP `npm install`
  created a package.json + node_modules _in the wrong repo_ and the gate
  scripts "vanished" ("Missing script: test:unit"). Diagnosed by `git log`
  showing the wrong repo's commits; cleaned up with J's approval, deps
  reinstalled in the right repo, and the lesson recorded: side-task
  commands use subshells or absolute paths, and a sudden "missing scripts"
  smell means check `pwd` before hunting gremlins.
- Agent-brief hygiene carried forward from M4: every brief now pins git to
  read-only (no restore/checkout/stash, not just no commits) and requires
  `${pipestatus[1]}` for piped gate exit codes. Both rules earned their
  keep this session — no repeats of either failure class.

## Verify

- **The perf gate earned its keep, then taught a measurement lesson.** The
  first §9.3 capture (4× CPU throttle) failed the assembly opening on three
  frame bursts mapped to the tokens and shell beats. Three targeted tunes
  followed: entrance offsets so few tweens start on the same frames as
  GSAP's mount work, the hero settle shortened so it stops overlapping the
  facsimile crossfade, and shadowed facsimile layers dropping their
  box-shadow on the live stage. Then the failing target started rotating
  run-to-run — assembly, rail, palette — each by exactly one frame at
  17.6–17.8ms worst, never near a ~33ms dropped-frame double. Headless
  Chromium's rAF is not vsync-locked; its idle cadence drifts to ~17ms, so
  a strict >16.7ms rule grades normal headless timing as failure. The
  script now runs median-of-3 per target and verdicts on genuinely missed
  frames (>25ms), reporting strict-budget stats alongside: **zero missed
  frames anywhere, all three targets PASS**. The rail also lost its
  per-scroll-frame React render (direct DOM transform writes) — real
  main-thread work the capture surfaced. The manual DevTools protocol
  remains the arbiter at the M6 design checkpoint.
- **The strict axe scan caught a real palette gap twice over.** The beat
  numerals in accent read 3.79:1 on the surface — under AA for 14px text —
  and the first fix (ink-muted) still only reached 4.24:1. The verify
  agent had carved the decorative spans out of the scan with a WCAG 1.4.3
  rationale; integration reversed that — the numerals moved to full ink and
  the scan runs strict, no exclusions. The finding that `--color-ink-muted`
  is sub-AA for small text on the base surface goes to the M6 design
  checkpoint as a token-level question.
- Full suite at close: **560 unit** · **84 request specs** · **94 e2e** —
  including the keyboard story (complete and skip variants, visible focus
  asserted at every stop), beat mechanics at five scroll fractions, the
  structural content-parity proof (figcaption sets identical across modes),
  the network proof (reduced-motion sessions never request the GSAP chunk;
  motion sessions do), and axe across both modes and both skins with no
  exclusions. Token discipline, RuboCop/ESLint/stylelint/Prettier/tsc,
  manifest drift gate, leak gate: all green.
