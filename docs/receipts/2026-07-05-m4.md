# M4 — dual-mode shell (2026-07-05)

Fifth build session. Goal: wire the already-built DS Nav (bar + ⌘K palette)
into a real site shell around every page — story mode at `/` with a fixed
"Skip to the work →" escape hatch, the 90-second skim hub at `/work`, chapter
route scaffolds, mode memory as a courtesy (never a redirect), and an on-brand 404. The shell is the milestone; no component got rebuilt.

## Session shape

- One frontier orchestrator + four mid-tier build agents + one small-tier 404
  agent in a single parallel wave. Contracts first, again: a shell README
  pinning every interface (SiteShell layout pattern, mode-memory semantics,
  the full palette action inventory as a table with stable ids, escape-hatch
  and skim-hub design specs) landed before any agent launched, along with all
  Rails routes and controllers — so five agents never touched a shared file.
- The e2e wave never got its agent: the mid-tier verify agent died on a
  server-side API error after twenty minutes of discovery (nothing written to
  disk), and its transcript-resume stalled out ten minutes later. After two
  infrastructure deaths the orchestrator wrote the verify suite inline
  instead — which turned out to matter, because the suite immediately caught
  two real app defects an agent might have papered over (below).
- Two deliberate pre-lands kept the parallel wave collision-free: the
  orchestrator implemented Nav's new `linkAs` prop directly (the actions
  agent only documented and tested it), and wrote a placeholder `actions.ts`
  implementing the frozen signature so the shell agent could compile before
  the real inventory existed. The real module replaced the placeholder
  wholesale — same move that made M1's generated-artifact bootstrap work.
- Model tiers per the spec's M4 row: skim-hub layout designed at frontier
  tier as a written spec (block order, type mapping, spacing rhythm), built
  at mid tier; the 404 ran small-tier against an exact composition brief.

## Task log

- **A mid-wave revert, survived.** Partway through the parallel wave,
  every orchestrator wave-0 edit to _tracked_ files vanished — routes,
  two controllers, and Nav's `linkAs` — while all agent-created untracked
  files survived, and agents kept landing edits on top of the now-reverted
  base. Likely a stray `git restore` from one agent's baseline-diffing
  habit; no agent reported running one. Detected because a finished agent's
  request specs depended on the reverted routes; re-landed from the session
  log in minutes, then verified the whole triangle (component ↔ meta ↔
  manifest) before integration. Lesson recorded: parallel agents get told
  not to _commit_; they now also need telling not to _restore_.
- **The landmark-name collision was designed away before it existed.** The
  nav component's doc page already mounts two Navs (variants demo +
  playground stage). The real shell adds a third. The playground's label
  control defaults to the API default "Primary" — so the shell takes
  `label="Site"` instead, and all three landmarks stay unique with the
  playground control still live (it was M3's one genuinely meaningful
  control; killing it was not an option).
- **Hidden means hidden, even in the palette.** The verify step says "switch
  skin via palette," but v1 ships one visible skin — the debug torture skin
  is a diagnostic, not an affordance, and exposing it would reopen a locked
  M1 decision. Resolution: the palette lists non-hidden skins only, and the
  e2e enters on `?skin=debug` then switches _back_ to galenti through the
  palette — a real, observable switch that leaves the design decision
  intact. M6's rails-era joins the palette by registry, zero shell edits.
- **Mode memory that can't gaslight.** Last-used mode is captured once at
  shell mount, _before_ the first write — so the hatch label ("Skip to the
  work →" vs "Continue to the work →") is cross-session memory that never
  flips mid-visit while you navigate. Neutral routes (system, resume, 404)
  write nothing. Two agents implemented the same three-line path check
  independently (one couldn't see the other's not-yet-written module);
  integration consolidated to one export and tightened the segment
  matching so an unmatched `/workshop` 404 can't pollute the stored mode.
- Review-as-you-go catches at integration: the story agent invented
  overlapping chapter era ranges (Ch1 ends 2019, Ch2 began 2018 — fixed to a
  clean boundary); dead skip-link CSS lingered after the DocShell dedupe;
  the shell agent's interim `linkAs` cast dissolved once the prop was
  confirmed on the component.
- The token-discipline lint bit an agent in a new way: the deny-list greps
  literal `--raw-` substrings _including inside CSS comments_ — a design
  note reading "never --raw-*" was itself a violation. Reworded, noted.
- **The axe matrix caught the shell floating outside the page's landmark
  structure.** The escape hatch — a fixed pill rendered between header and
  main — violated the region rule ("content not contained by landmarks") on
  every story page. The fix is semantic, not cosmetic: the hatch is
  navigation, so it got a labeled `<nav>` wrapper (unique name "Escape
  hatch"), and the transient toast got a labeled `<aside>` for the same
  reason on open-state scans.
- **The 404 forced a second component API addition, same pattern as M3's Nav
  `label`.** EmptyState rendered its title as a `<p>` by design ("consumers
  own heading structure") — but a full-page 404 built from it then has no h1
  at all, which fails axe and leaves AT users with an unlabeled page. The
  honest fix was `titleAs` (default `p`, the 404 passes `h1`): the consumer
  still owns heading semantics, now with an API to express them. Drift gate
  forced component, `meta.ts`, and manifest to move together again.
- e2e integration lessons: the global ⌘K listener attaches after client
  hydration, so the spec's palette opener retries the shortcut until the
  dialog answers (a too-early press is a no-op); a fixed-position toast makes
  its landmark wrapper zero-size, so visibility asserts target the
  `role="status"` element, not the wrapper — the wrapper-based assert passed
  in one test and failed in another purely on which matcher it used; the
  clipboard test that exists to scan the toast's open state stubs
  `writeText` (headless focus quirks), while the dedicated copy-email test
  reads the real clipboard back.

## Verify

- Milestone verify held, keyboard-only: land on `/` (hatch visible in the
  first viewport), reach `/work`, open the palette with the keyboard, switch
  skin debug → galenti through it. Skim hub reachable in ≤2 interactions
  from story, system, and 404 contexts. On-brand 404 served with a real 404
  status from the Rails catch-all, `Manifest::NotFound` upgraded from a bare
  `head :not_found` to the same branded page.
- Reduced-motion proven end-to-end, not assumed: e2e emulates
  `prefers-reduced-motion`, asserts the hatch's transition collapses to 0s
  (against a non-reduced control run proving the collapse is real), the
  palette stays functional, and the chapter reading-progress rail still
  tracks position — position is state, not motion.
- Full suite at close: **523 unit** · **84 request specs** · **84 e2e** —
  including the new-route axe matrix in both skins, the three-nav
  landmark-uniqueness proof on the nav doc page, and the toast open-state
  scan. Token discipline, RuboCop/ESLint/stylelint/Prettier/tsc, manifest
  drift gate (16 components, no drift), leak gate: all green locally.
- Visual F-review on screenshots: the chapter progress rail stacks cleanly
  above the sticky nav; the hatch floats over gateway content at narrow
  widths like any fixed affordance (raised surface keeps it legible — and
  M5's assembly opening pushes the gateway below the fold anyway).
