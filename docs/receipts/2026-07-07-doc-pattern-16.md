# 2026-07-07 — Full doc pattern across all 16 (v1.5)

## Goal

Promote the 11 gallery-tier components to the same doc-page pattern the hero 5
carry. Survey finding that shaped the session: the doc page already renders
every section (variants, props, tokens, a11y, usage, code, skin switcher) for
every tier — the only genuinely hero-gated feature was the props playground,
discovered by glob on `playground.tsx`. So "full doc pattern" collapsed to:
eight playground hosts, two manifest prop opt-outs, a tier-neutral copy sweep,
and e2e walks. Three components stay variants-only by design: Prose
(children-only API), Skin Switcher (its demo IS the live component), Tabs
(only invisible/fragile text controls possible).

## Fleet

Frontier orchestrator + 3 parallel sonnet agents against pinned contracts:

| Agent | Task | Output |
|---|---|---|
| hosts-1 [M] | Badge, Card, CodeBlock, EmptyState hosts + `code-block.yml` `playground: false` on `code` | 4 files + 1 edit |
| hosts-2 [M] | Menu, Skeleton, Toast, Tooltip hosts + `toast.yml` `playground: false` on `open` | 4 files + 1 edit |
| sweep [M] | Tier-neutral copy across doc page / registry / contract / READMEs + 3 e2e playground walks | 6 edits + 3 tests |

Files were fully disjoint across agents — no worktrees needed. Cross-agent
coordination was done by pinning exact strings in the briefs before launch:
the e2e agent wrote walks against contract strings (`Show toast`, `Saved`,
`Focus me`, `More info`) while the host agent built to them in parallel; when
both landed, the blind-written selectors matched the real DOM exactly.

## Workflow notes

- The Dialog host was the template for controlled-open components (Toast) and
  required-string fallbacks (EmptyState, Menu label, Tooltip content); Button
  was the template for pure spread hosts (Badge, Skeleton, Card).
- Sweep agent caught a consequence the briefs missed: the old Figma fallback
  copy was asserted in `show.test.tsx` — updated alongside the copy change
  instead of leaving a failing test for the verify step to find.
- Toast renders at the real viewport corner from the playground stage (host
  trigger like Dialog) — honest behavior over CSS containment hacks.
- `tier` semantics deliberately unchanged: it now means flagship curation
  (index/DocShell grouping), not doc completeness. Zero manifest tier churn.
- No `number` control added to the generator this pass — Skeleton `lines` and
  Toast `autoHideMs` stay auto-skipped (deliberate deferral).
