# M0 — repo bootstrap (2026-07-04)

First build session. Goal: public repo skeleton — Rails 8.1, Inertia, React
19/TS, and Vite, plus three test harnesses, a lint wall, the leak-check gate,
CI, and the deploy blueprint.

## Session shape

- Single orchestrator session (frontier-class model, Claude Fable 5), no
  subagents — bootstrap is inherently serial (toolchain → scaffold → harnesses
  → verify), so there was nothing to parallelize. Later milestones fan out.
- Human gates honored: first public push and hosting setup held for explicit
  owner approval.

## Task log

- Toolchain from zero: `gh` + `mise` via Homebrew; Ruby 3.4.10 + Node 22.23.1
  pinned per-repo. Ruby compile failed once on a missing libyaml (psych
  extension) — fixed and recompiled.
- Dependency pins resolved against live registries. Two ecosystem drifts
  handled: latest Vite is 8.x and TypeScript 6.x, but this repo pins Vite 7.x
  / TS 5.9.x — which forced `@vitejs/plugin-react` down to 5.2.0 (v6 requires
  Vite 8). The Inertia installer pulled the newer majors; downgraded post-install.
- `rails new` with a lean flag set (postgres, no hotwire/solid/kamal/docker,
  minitest skipped in favor of RSpec), then `inertia:install --framework=react
--typescript --vite --no-tailwind`.
- Gotcha: inertia_rails auto-derives component names as
  `controller_path/action` (`pages/home`), while the frontend convention is
  `home/index.tsx` — fixed with an explicit component name in the controller.
- Gotcha: `git grep` skips untracked files, so the leak-check gate scanned
  nothing pre-first-commit — added `--untracked`. Gate was then self-tested
  with a planted leak (caught it) before being trusted.
- Axe caught a real violation on the first e2e run: missing `lang` on
  `<html>`. Fixed in the layout.

## Verify (all green locally)

leakcheck · RuboCop (26 files, 0 offenses) · ESLint · Prettier · stylelint ·
tsc ×2 configs · RSpec 2/2 · Vitest 1/1 · Playwright 2/2 (smoke + axe scan,
real chromium against a booted test server).

## Landing (same day)

- First push approved and made; remote CI green after one fixup (Prettier on
  this very file — the gate caught its own receipts).
- First deploy failed twice, both real catches:
  1. The production bundle contained a 452 kB **test-file chunk** — Inertia's
     `pages: '../pages'` glob sweeps up `*.test.tsx`. Replaced with an
     explicit resolver whose glob excludes test files.
  2. `rake assets:clean` doesn't exist without the asset pipeline, and the
     host's Rails auto-detection invokes it after every build. Provided a
     no-op task.
- Deploy green on the third run; staging serves the hello-world page with
  a 200 health check. Milestone verify complete: remote CI green (incl. leak
  gate) + staging serving the Inertia page + zero private references.
