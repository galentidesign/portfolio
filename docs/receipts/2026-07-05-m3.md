# M3 — manifest + DS doc pages (2026-07-05)

Fourth build session. Goal: make `/system` a pure function of
`data/manifest/*.yml` — one YAML per component, a Rails model that validates
it at boot, a CI drift gate that cross-references it against each component's
`meta.ts` and CSS, and doc pages (with live props playgrounds on the hero
five) generated from it. Delete a manifest file and the build fails; that
governance gate is itself the exhibit.

## Session shape

- One frontier orchestrator + five mid-tier build agents + sixteen small-tier
  manifest writers, in overlapping waves. Contracts first, again: the manifest
  schema README + a hand-written `button.yml` exemplar, the playground host
  contract, and the DocShell layout all landed before any agent launched — the
  same pin-the-interfaces move that made M1/M2 parallelism rework-free.
- The five mid-tier tasks split by boundary, not by page: Rails model+routes,
  drift gate, system pages, doc-page template + playground machinery, and the
  five hero playground hosts. Disjoint file ownership; the two page agents
  consumed the DocShell the orchestrator had already written rather than
  stubbing each other.
- The sixteen manifest writers ran one-component-per-agent against the schema
  README and exemplar, with the token list produced by a pinned shell
  one-liner (grep the component's own `var(--…)` references) instead of
  judgment — the drift gate later confirmed all sixteen lists exactly,
  first try.
- Eleven gallery manifests launched while the mid-tier wave was still
  building; the four hero manifests waited for the host agent's report of
  which props are host-managed (`playground: false`).

## Task log

- **Review-as-you-go catches, hero-host wave:** the Table host invented
  literal token values as demo data (plausible numbers for real token names —
  exactly the kind of quiet dishonesty a DS docs site can't carry); swapped
  for structural inventory rows that can't drift. The Dialog host styled its
  demo body with inline `style={{…}}` instead of dogfooding Prose.
- **A mid-flight brief update that landed:** reviewing the Nav host surfaced
  that the doc page mounts two Navs (variants demo + playground stage), so
  two global ⌘K listeners would fight. The fix (stage pins
  `enableShortcut={false}`, manifest opts the prop out of controls) was
  messaged to the nav manifest agent while it was still writing — the flag
  was in its YAML when it reported.
- **The YAML colon-space trap:** two manifests had prose list items
  containing `: ` (`Roving focus: …`, `(pointer-events: none)`), which YAML
  parses as single-key hashes. The Rails model's strict
  `aria: Array of Strings` validation caught both — found by the drift-gate
  agent smoke-testing the other agent's model mid-session, fixed by quoting,
  and now warned about in the schema README. Two independent agents reported
  the same defect from opposite sides of the contract.
- **The axe matrix caught a real component API gap.** The nav doc page's two
  Navs both rendered `<nav aria-label="Primary">` — `landmark-unique`
  violation. The correct fix was not a workaround but an API addition: Nav
  grew a `label` prop (the APG unique-landmark requirement any real app with
  two navs hits). The drift gate then forced component, `meta.ts`, and
  manifest to move together — and the new prop became the Nav playground's
  one genuinely meaningful control: renaming a landmark live on the page.
- The request specs originally isolated themselves behind a fixture copy of
  `button.yml` (a reasonable dodge while two manifests were broken);
  integration reverted them to run against the real tree — serving the
  checked-in manifest end-to-end is exactly the milestone's claim, and a
  fixture copy is one more thing that can silently go stale.
- e2e integration finds: sr-only radio inputs can't be `click()`ed
  (the label is the click surface, as for real users); a page hosting two
  palette triggers needs region-scoped locators; the M1 tokens-page e2e
  assertions (skin links, `main` background) were rewritten for the DocShell
  rehome rather than deleted.
- Tooling notes: the token-optimizing CLI wrapper masked a prettier failure
  (success prose, exit 1) — caught because the session verified exit codes
  explicitly; ruby needed explicit version pinning in orchestrator shells
  (returning M1 gotcha).

## Verify

- `rake manifest:verify` — **16 components verified, no drift**, both
  directions proven: deleting a manifest file fails the gate naming the
  missing slug; restoring returns it to green. The gate runs in CI's lint job
  with no database.
- Full suite at close: **404 unit tests** · **77 request/model specs** ·
  **57 e2e** — including the milestone matrix: all sixteen doc pages ×
  both skins × zero axe violations, open-state scans (dialog open, palette
  open), keyboard walks (⌘K → type → Enter performs; Escape returns focus;
  skip link; SkinSwitcher radios reachable with visible focus), and the
  playground proof: click a segment → the live component re-renders with the
  new prop and the code snippet updates.
- Manifest prose line-edited at review: implementer-voice guidance ("use
  roving tabindex…") rewritten as consumer guidance; four files touched.
- Token discipline, RuboCop/ESLint/Prettier/tsc ×2, leak gate: all green.

## Landing

- 97 files changed, ~7,650 insertions, staged as five logical commits:
  manifest contract + sixteen YAMLs · Rails manifest engine + routes · CI
  drift gate · manifest-driven docs UI + playgrounds · this receipt.
- The scratch `/system/gallery` page is gone; its glob-discovered demos now
