# M2 — DS components, all sixteen (2026-07-04 → 05)

Third build session, continuing directly from the M1 design checkpoint. Goal:
the full component inventory — hero 5 (Button, Form Field, Table, Dialog,
Nav with the ⌘K palette) built a11y-first, gallery 11 stamped from a shared
contract, every one a pure function of the token engine.

## Session shape

- One frontier orchestrator + sixteen component subagents in two waves, one
  component per agent, each owning exactly one directory. The glob-discovered
  `gallery.tsx` convention exists precisely so sixteen parallel agents never
  touch a shared file.
- Contract-first again: `ds/components/README.md` (directory template,
  data-attribute variant convention, focus/motion/a11y ground rules, the
  meta.ts shape the M3 drift check will consume) plus per-component API
  pins written by the orchestrator before any agent launched. The API design
  is the frontier work; the implementation is mid-tier work against a
  contract.
- One deliberate tier deviation from the spec, approved at check-in: the
  keyboard-heavy gallery trio (Select/Menu, Tabs, Tooltip) ran on the
  mid tier instead of small — roving focus, typeahead, and dismissal
  semantics are where small-tier output produces retrofit a11y.
- Schema v1.1 rider: the component work surfaced a real token gap
  (small-control height), fixed the additive way — one new semantic key
  (`--density-control-sm`) in both skin files, parity-enforced, zero
  component workarounds.

## Task log

- Review-as-you-go beat review-at-the-end: each hero component got its
  frontier pass the moment its agent reported, while the others still built.
  Three real defects were caught this way, all invisible to the authors' own
  green test suites:
  - **Button** silently dropped a consumer's `onClick` in anchor mode (the
    handler was destructured out of the rest-props, then read back off the
    rest-props). Fixed inline + regression test.
  - **Dialog** dismissed on a text-selection drag that started inside the
    panel and released over the backdrop — the classic form-wipe hazard.
    Fixed with press-start tracking (gesture must begin AND end on backdrop).
  - **Nav palette** had a keyboard-order/visual-order divergence: option ids
    and Enter-activation followed filter order while rows rendered in group
    order, so interleaved groups made Enter perform a different action than
    the highlighted one. Bounced back to the owning agent with a fix list
    (plus: auto-highlight the top hit so type-then-Enter works, and two ARIA
    hygiene items); it returned 32/32 with a regression suite.
- Table and FormField passed review as built — the pinned per-component API
  briefs did their job.
- Five of the twelve concurrent agents stalled on a stream watchdog
  (infrastructure, not task failure) — two of them mid-verify with all files
  already on disk. All five resumed from their transcripts with a one-line
  continuation message and picked up exactly where they stopped; nothing was
  rebuilt that had already been written.

- Two more catches after the gallery-11 wave:
  - **Tabs** flagged its own hazard back at the orchestrator's pinned brief:
    every tab carried `aria-controls` while only the selected panel was
    mounted — dangling idrefs the axe matrix would reject. The brief was the
    bug; the fix (all panels mounted, inactive ones `hidden`) went back to
    the owning agent.
  - **Toast** could stack two live auto-hide timers through interleaved
    hover/focus resumes (double dismiss); fixed inline at review.
- Cross-cutting finds the component agents handed the session for free: RTL
  16 routes user-event through an async wrapper that only advances _jest_
  fake timers — under vitest every `user.*` call hangs unless a scoped
  `jest.advanceTimersByTime` shim bridges to `vi.advanceTimersByTime`. Two
  agents hit it independently; the second confirmed the first's fix pattern.
- The e2e axe matrix earned its keep on the first real run: one violation,
  `landmark-unique` — CodeBlock's keyboard-scrollable `<pre>` used
  `role="region"`, which mints a LANDMARK per code block; unlabeled blocks
  all announce as 'Code'. Swapped to `role="group"` (focusable + named, no
  landmark pollution). The other two e2e failures were the orchestrator's
  own spec racing hydration — keypresses before React mounted; fixed with a
  demo-count readiness anchor that doubles as a hydration barrier.
