# M1 — token engine + `galenti` skin (2026-07-04)

Second build session. Goal: the crown-jewel subsystem — token JSON → compiled
skin CSS + typed registry, the reduced-motion gate, the semantic-token lint
wall, and the first real skin (`galenti`, warm precision light) with a hidden
`debug` torture-test skin proving the additive-skin contract.

## Session shape

- One frontier orchestrator (Claude Fable 5) + six subagents across two waves.
  Frontier work stayed in-session: token schema design, the full `galenti`
  palette/type/motion values (WCAG-audited numerically before a single file
  was generated), agent briefs, integration, and the cross-cutting review.
- Wave 1 ran five agents in parallel on disjoint file sets — no worktrees
  needed because the contract (`ds/tokens/README.md`) was written first and
  every brief carved exclusive file ownership: fonts [haiku] · build engine
  [sonnet] · Rails plumbing [sonnet] · providers [sonnet] · lint rule
  [sonnet]. Wave 2 was one sonnet for the `/system/tokens` specimen page +
  e2e, launched only after the orchestrator had integrated and re-verified
  wave 1.
- Two shared-file collision risks were pre-empted by the orchestrator editing
  those files up front (`package.json` script names) or reserving them for
  integration (`ci.yml`, the layout's entry-tag order).

## Task log

- Contract-first paid off: three of the five wave-1 agents needed the
  generated artifacts before the generator existed. Because the README pinned
  the exact `skins.ts`/`skins.json` interfaces, each agent bootstrapped a
  temporary copy by hand and the real build later overwrote them with
  byte-compatible output. Zero rework.
- Parallel-agent friction was real but shallow: one agent patched two other
  agents' files with `@ts-nocheck` to get its own typecheck green (the
  missing `@types/node` landed seconds later from a sibling). Integration
  removed every band-aid and re-verified strict TS clean.
- The one genuine integration gap was the orchestrator's own brief bug:
  nobody was told to import the generated CSS into an entrypoint, so the
  first production build shipped zero styles. Caught by inspecting the build
  manifest during integration — the token payload for all skins compiles to
  7.4 kB (1.9 kB gzip).
- WCAG math ran before design values were committed: 4 of 18 candidate
  `galenti` pairs failed the first numeric audit (faint ink, accent-on-wash,
  caution pair, strong hairline on sunken wells) and were re-tuned to pass
  with margin. The `debug` skin passed all 12 pairs first try — garish and
  compliant are not opposites.
- The `debug` skin is deliberately hidden from future switcher UIs
  (`hidden: true` in the registry) but always reachable via `?skin=debug` —
  a permanent, public torture test: any surface that ignores the engine
  shows up magenta-on-void.
- mise-managed Ruby needed explicit pinning in orchestrator shells
  (`mise exec ruby@3.4.10 --`) — agent shells resolved it fine.

## Verify

Full suite locally: leakcheck · RuboCop · ESLint · Prettier · stylelint
(incl. the new `portfolio/no-raw-tokens` rule) · `lint:tokens` sweep ·
tsc ×2 · Vitest · RSpec · Playwright (+axe on both skins) · production Vite
build. Lint teeth were proven by planting raw-token leaks in CSS and TSX and
watching both gates fail, then removing the plants. The milestone verify —
token build in CI, `/system/tokens` rendering `galenti` and the stub skin via
URL param — went green before the design checkpoint.
