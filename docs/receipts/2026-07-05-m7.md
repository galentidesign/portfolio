# M7 — Figma library parity (2026-07-05)

Eighth build session. Goal: the design system becomes a twin public artifact
— the same token JSON that compiles to CSS custom properties now also drives
a Figma variable collection with one mode per skin, and the hero five exist
as true Figma component sets bound to it. Mode switch in Figma mirrors the
site's skin switch.

## Session shape

- One frontier orchestrator driving the Figma MCP inline, with one read-only
  extraction agent. The canvas is a single shared file, so the component
  loop ran sequentially by design (parallel canvas writers were ruled out at
  plan time); the extraction agent ran concurrently up front and returned a
  per-component brief — variant axes, anatomy, exact token-consumption maps,
  states, and fixed literals — that the whole build consumed.
- **The collection mirrors the two-layer token architecture, not just the
  values.** Three collections: `Primitives` (36 raw colors from both skins,
  hidden scopes), `Tokens` (65 semantic variables, modes `galenti` +
  `rails-era` — colors alias primitives per mode exactly as the CSS build
  resolves `{raw.color.*}` refs), and `Base` (18 space/z floats). Every
  variable carries explicit scopes and WEB code syntax matching its CSS
  custom-property name. Viewport-dependent `clamp()` values resolve at a
  documented 1440px reference; `density/gap` aliases the Base spacing scale
  cross-collection, as the code does.
- **A bindability experiment beat the reference docs.** The MCP skill's
  variable reference says `fontSize`/`fontWeight`/`lineHeight` can't bind via
  the plugin API; a probe showed fontSize, fontWeight, AND fontFamily all
  bind — and render, including galenti's variable-font weights (640/620/600)
  that have no named instances. The entire type ramp mode-switches: body
  text flips Hanken Grotesk 17px ↔ Source Sans 3 14px with the collection
  mode. Shadows are the one thing that can't (effect styles have no modes)
  — four styles ship (`raised`/`overlay` × skin) with the limitation noted
  in component descriptions.

## Task log

- Hero five built as bound component sets, each with mode-pinned preview
  pairs: **Button** (24 variants — Variant × Size × State, Label text prop,
  Icon-start boolean + INSTANCE_SWAP pointed at a three-glyph icon set,
  Busy spinner), **Form Field** (8 — Kind × State, required/hint/value
  props), **Table** (single, sortable header + density-bound rows),
  **Dialog** (sm/md, dismissible/description/footer booleans — the footer
  holds real Button instances that mode-flip inside it), **Nav** plus a
  separate **Nav / Palette** (bar with current-link accent underline and
  ⌘K chip; palette with active option). Foundations page carries the
  19-swatch semantic color grid and type specimens, each rendered twice
  from ONE bound structure under two mode pins.
- **Two Figma-side defects caught by screenshot validation, both real.**
  A two-paint fill stack (bound accent under an 8% black overlay) renders
  solid black despite correct node data — restructured as an absolute
  overlay child, which is also more honest to the CSS (`inset 0 0 0 999px`
  IS an overlay layer). And `query('INSTANCE[name=Icon start]')` matches
  nothing silently — spaces in unquoted attribute selectors fail without
  erroring, so the icon slot's property references never linked and the
  swap menu was dead; refs re-linked via `children.find` across all 24
  variants.
- Two font gotchas: the `↑` sort glyph resolves to a symbols-fallback
  family that must be loaded before any bound-variable write touches the
  node, and every `use_figma` call is a fresh plugin context — font loads
  never persist across calls.
- Back-reference comments landed in all five component sources (file key,
  node URL, prop→axis mapping — the Pro-plan stand-in for Code Connect),
  and the five manifest YAMLs got their real `links.figma` URLs. The
  manifest drift gate held throughout; doc pages now render the live
  library links.
- The wrong-ruby trap resurfaced through a new door: playwright's own
  `webServer` boot (config command runs un-pinned). Booting the server
  manually under mise and letting `reuseExistingServer` pick it up is the
  session-local workaround; the config stays untouched for CI, where ruby
  is provisioned.
